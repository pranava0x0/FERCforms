🎓 LESSONS LEARNED
═════════════════════════════════════════════════════════════════════════════

1. State PUCs have NO standard website structure
   → Requires per-state parsers (expensive but necessary)

2. Landing pages ≠ documents
   → Most state sites show portals, not direct PDF links
   → Solution: Focus on direct URLs first, build portal scrapers later

3. Inline > Workflow for bounded, sequential tasks
   → Token cost dominates latency for small fan-outs
   → Parallelism irrelevant when rate-limited

4. Ingest first, parse second
   → Better to have structured metadata without findings
   → Than findings parser that doesn't understand state formats
