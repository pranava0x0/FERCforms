  - **`fmtAmount` rendered `$1000K` instead of `$1M`** at values that round up across a unit boundary (999,999). **Fix:** unit thresholds lowered 0.5% so a rounding-up value lands in the higher unit; also fixed negative-sign placement (`-$5M`, not `$-5M`). Display-only.

Root cause of the 4 real bugs: **code bug** in this session's new `--live` check + UI.
